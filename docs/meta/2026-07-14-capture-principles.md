# Принципы фиксации

Category: Principle

Этот документ содержит принципы, выявленные в процессе разработки TrailsLog.

Это не полный список. По мере развития проекта здесь могут появляться новые принципы, а существующие — уточняться.

## Текущие принципы

### Фиксация должна требовать минимальных усилий

В идеале для фиксации выполненного действия должно быть достаточно одного простого действия пользователя.

### Фиксация не должна мешать самой деятельности

После записи пользователь должен практически сразу иметь возможность продолжить то, чем занимался.

### Добавление записи должно занимать меньше усилий, чем решение её не делать

Если фиксация становится сложнее или требует слишком много внимания, значительная часть ценных наблюдений никогда не будет записана.

## Следствия

Эти принципы используются при принятии проектных решений.

Например:

- команды должны оставаться максимально простыми;
- дополнительная информация должна быть необязательной;
- интерпретация должна происходить позже;
- сырые данные должны сохраняться без потерь;
- внутренние структуры данных могут изменяться без изменения процесса фиксации.
