# Как Git стал частью системы сопровождения знаний

Category: Observation

Knowledge type: Knowledge acquisition

Во время обсуждения сопровождения документов `meta` возник вопрос.

Как сохранять историю формирования знаний?

Первоначально рассматривались различные варианты.

Можно было вводить версии документов.

Можно было создавать журналы изменений.

Можно было хранить специальные ревизии знаний.

Однако постепенно внимание переключилось на Git.

Изначально Git воспринимался исключительно как технический инструмент хранения истории изменений файлов.

Но при разборе нескольких реальных примеров неожиданно выяснилось, что в некоторых случаях история изменений документа одновременно оказывается историей формирования знания.

Это наблюдение привело к следующему вопросу.

Всегда ли этого достаточно?

Разбор последующих примеров показал, что ответ отрицательный.

Если происхождение изменения естественно восстанавливается по самому изменению документа, Git уже сохраняет историю появления знания.

Если же происхождение изменения требует самостоятельного объяснения, появляется отдельный документ, описывающий путь появления этого знания.

Таким образом возникла идея использовать Git не самостоятельно, а совместно с методикой сопровождения знаний.
