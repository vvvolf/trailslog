# Git как механизм сопровождения истории знаний

Category: Principle

Во время сопровождения знаний возникает две разные задачи.

Первая — сохранить текущее состояние знания.

Вторая — сохранить историю его формирования.

На первый взгляд кажется, что для решения второй задачи требуется специальная система управления знаниями.

Однако в ходе разработки TrailsLog была выдвинута следующая рабочая гипотеза.

> При соблюдении определённой методики Git может использоваться как механизм сопровождения истории формирования знаний.

## Почему это возможно

Сам по себе Git не хранит знания.

Он хранит только историю изменений документов.

Однако знания в проекте фиксируются именно через документы.

Поэтому изменение документа часто одновременно является изменением знания.

Если происхождение изменения естественно восстанавливается по самому изменению документа, история Git становится достаточной для восстановления истории появления этого знания.

Именно в таких случаях дополнительная фиксация происхождения знания не требуется.

## Ограничение

Так происходит не всегда.

Иногда изменение документа невозможно естественно понять только по самому изменению.

Попытка объяснить его происхождение превращается в самостоятельное рассуждение.

В этом случае одного Git уже недостаточно.

Однако это не означает отказ от использования Git.

Недостающая часть истории оформляется отдельным документом.

После этого история формирования знания снова оказывается представлена в Git — уже через новый документ.

Таким образом Git используется совместно с методикой сопровождения знаний, а не самостоятельно.

## Методика

На сегодняшний день предполагается следующий подход.

Если происхождение нового знания естественно восстанавливается по истории изменений документа, достаточно Git.

Если происхождение знания требует самостоятельного объяснения, создаётся отдельный документ, описывающий ход появления этого знания.

Таким образом сохраняется баланс между простотой сопровождения проекта и прозрачностью формирования знаний.

## Единицы сопровождения знаний

В процессе дальнейшей работы стало понятно, что принцип смысловой целостности относится не только к коммитам.

Однако речь идёт не о знаниях вообще.

Любое достаточно большое знание естественным образом состоит из множества частей.

Напротив, в процессе исследования постоянно возникают небольшие самостоятельные единицы понимания.

Именно история появления таких единиц становится минимальным объектом сопровождения.

Если история появления одной такой единицы оказывается искусственно разделённой между несколькими независимыми коммитами или несколькими документами `Knowledge acquisition`, значительно усложняется восстановление того, каким образом данная единица понимания возникла.

Формально вся информация может сохраниться.

Однако сама единица понимания перестаёт существовать как самостоятельный объект сопровождения.

### Коммит

Коммит рассматривается не как набор изменений файлов.

Коммит фиксирует появление одной законченной единицы исследования во времени.

Если для выражения этой единицы требуется изменить несколько документов, они должны входить в один коммит.

Если же в одном коммите оказываются несколько независимых единиц понимания, восстановление истории их формирования существенно усложняется.

### Документ `Knowledge acquisition`

Документы данного типа используются для сопровождения процесса исследования.

Каждый такой документ рекомендуется строить вокруг одной самостоятельной единицы исследования.

Это позволяет использовать сами документы как минимальные элементы истории формирования знаний.

При этом подобное ограничение относится только к документам `Knowledge acquisition`.

Документы, описывающие уже сформировавшуюся систему знаний, могут естественным образом содержать множество связанных между собой знаний, организованных с помощью разделов, подразделов и других средств структурирования текста.

### Общий принцип

Коммиты и документы проекта являются средствами сопровождения процесса исследования.

При этом они выполняют разные функции и не обязаны соответствовать друг другу.

Коммит фиксирует момент появления новой единицы исследования.

Документ `Knowledge acquisition` сохраняет её внутреннюю историю.

Требование смысловой целостности относится не к коммитам и не к документам как таковым.

Оно относится к самой единице понимания.

История формирования одной самостоятельной единицы понимания не должна искусственно разрываться между несколькими независимыми коммитами или несколькими документами `Knowledge acquisition`.

При этом одна такая единица может потребовать изменения нескольких документов внутри одного коммита.

И наоборот, один документ может со временем участвовать в формировании нескольких различных единиц понимания, если они возникают последовательно и фиксируются отдельными изменениями.

Таким образом объектом сопровождения является не документ и не коммит сами по себе.

Объектом сопровождения является история формирования самостоятельной единицы понимания.

Коммиты и документы лишь фиксируют различные стороны этой истории.

## Стоимость сопровождения исследования

Фиксация процесса появления знаний значительно дороже, чем фиксация самих знаний.

Именно поэтому документы `Knowledge acquisition` рекомендуется делать максимально небольшими и посвящёнными одной самостоятельной единице исследования.

Это уменьшает стоимость дальнейшего сопровождения, повторного чтения и повторного использования накопленного материала.

Одновременно документы, описывающие уже сформировавшуюся систему знаний, могут постепенно агрегировать множество подобных единиц, формируя более крупные структуры знаний.

Тем самым достигается баланс между подробностью сопровождения исследования и удобством использования накопленных знаний.

## Статус

Данный подход рассматривается как рабочая гипотеза.

Она возникла в процессе разработки самой системы управления знаниями и ещё должна быть проверена практикой по мере роста проекта.
