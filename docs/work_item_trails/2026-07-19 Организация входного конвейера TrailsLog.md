resume_context: |
    # Восстановление контекста: Организация входного конвейера TrailsLog
    
    Контекст относится не ко всему проекту TrailsLog, а только к обсуждению организации входного конвейера необработанных материалов (руды) и связанных с этим архитектурных решений.
    
    Во время обсуждения были сформулированы следующие идеи.

    ## Исходная проблема

    Возникла повторяющаяся ситуация, когда в длинных диалогах появляются важные мысли, которые невозможно сразу осмыслить, но не хочется потерять.

    Использование `meta` для хранения подобных материалов признано неподходящим, поскольку `meta` предназначена для хранения знаний, а не необработанных материалов.

    Отсюда возникла идея отдельного входного конвейера.

    ## Основная гипотеза

    Входной конвейер должен хранить не знания, а кандидатов на дальнейшее осмысление.

    Первоначально обсуждалось название _Work Item_, однако позже более естественным было признано название **Inbox**.

    Inbox рассматривается как очередь необработанных элементов, ожидающих дальнейшей обработки.

    ## Структура элемента Inbox

    Каждый элемент может сопровождаться Markdown-файлом с YAML-метаданными.

    При этом обсуждалась возможность использовать уже существующие принципы проекта вместо введения новых сущностей.

    Главной идеей стало повторное использование концепции `scope_of_understanding`.

    В ходе обсуждения также возникла мысль, что для элементов Inbox может оказаться полезным отдельное поле, описывающее причину сохранения материала (обсуждались варианты `reason_for_capture`, `why_keep_this` и др.), однако окончательное решение пока не принято.

    ## Связь с Meta

    Inbox рассматривается как предшествующая стадия формирования знаний.

    После обработки материал либо:

    - превращается в документы `Knowledge acquisition`;
    - становится частью существующих документов знаний;
    - удаляется как не представляющий ценности.

    Таким образом Inbox является входным слоем процесса формирования знаний.

    ## Архитектурное решение

    Для эксперимента предложено создать каталог:

    ```
    work_items/
        inbox/
    ```

    внутри проекта TrailsLog.

    Этот Inbox предназначен только для исследования данной методики и документирования её разработки.

    Личные входные очереди предполагается хранить отдельно и не смешивать с экспериментальным Inbox проекта.

    ## Исследовательский вопрос

    Главный открытый вопрос состоит не в реализации Inbox, а в том, способен ли подобный входной конвейер стать универсальным способом сопровождения любых типов необработанных материалов:

    - текстов;
    - диалогов;
    - заметок;
    - папок с файлами;
    - архивов;
    - фотографий;
    - любых других объектов, из которых впоследствии могут извлекаться знания.

    ## Что следует обсуждать дальше

    Продолжить исследование архитектуры Inbox как универсального входного конвейера TrailsLog.

    Особое внимание уделить:

    - минимальному набору метаданных элемента Inbox;
    - связи Inbox с системой `Knowledge acquisition`;
    - взаимодействию Inbox с `scope_of_understanding`;
    - возможности автоматического построения обзора содержимого Inbox;
    - критериям перехода элемента из Inbox в систему знаний.
