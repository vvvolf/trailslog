# Work Item Specification

## Назначение

Work Item фиксирует самостоятельную единицу дальнейшей работы.

Он не является документом системы знаний.

Он существует для сопровождения процесса работы.

На текущем этапе развития методики настоящая спецификация описывает только минимальные требования к структуре Work Item.

Правила формирования содержимого продолжают исследоваться.

До выделения полной формальной спецификации эталонной реализацией считается документ `chatgpt_prompt.md`.

---

## Обязательные метаданные

```yaml
---
work_item_scope: |
    ...

reason_for_capture: |
    ...

resume_context: |
    ...
---
```

### work_item_scope

Определяет границы сопровождаемой единицы дальнейшей работы.

Должен отвечать на вопрос:

> Какую самостоятельную единицу дальнейшей работы сопровождает данный документ?

Поле должно описывать именно границы сопровождаемой единицы.

Оно не является кратким пересказом документа.

После чтения поля должно быть понятно:

- что относится к данной единице работы;
- что в неё сознательно не входит.

---

### reason_for_capture

Определяет причину сохранения.

Должен отвечать на вопрос:

> Почему данную тему необходимо сохранить как самостоятельную единицу дальнейшей работы?

Следует фиксировать именно причину сохранения.

Например:

- появилось новое понимание;
- возникла новая идея;
- обнаружена перспективная гипотеза;
- тема требует самостоятельной проработки;
- тема обладает самостоятельной практической ценностью.

---

### resume_context

Предназначен для восстановления контекста обсуждения при продолжении работы с ИИ.

Поле должно позволять открыть новый чат и практически сразу продолжить работу именно над данным Work Item без необходимости заново восстанавливать контекст из полного текста документа.

`resume_context` не является пересказом документа.

Желательно включать:

- что уже считается установленным;
- какие решения уже приняты;
- какие варианты уже отвергнуты;
- какие вопросы остаются открытыми;
- что наиболее естественно обсуждать следующим шагом.

---

## Рекомендуемая структура документа

После блока метаданных рекомендуется следующая структура.

```md
# Название

## Извлечённые реплики

...

## Заметки
```

### Заголовок

Заголовок должен отражать сопровождаемую единицу дальнейшей работы.

Тип документа в заголовок включать не следует.

---

### Извлечённые реплики

В данный раздел помещаются сообщения исходного обсуждения, относящиеся к сопровождаемой единице дальнейшей работы.

Рекомендуется:

- сохранять исходный порядок сообщений;
- не пересказывать их;
- не редактировать текст;
- использовать цитирование.

---

### Заметки

Раздел предназначен для дальнейшей ручной работы над Work Item.

Его содержимое не регламентируется настоящей спецификацией.

---

## Требования

Корректный Work Item должен удовлетворять следующим требованиям.

- сопровождает одну самостоятельную единицу дальнейшей работы;
- содержит достаточный контекст для возобновления обсуждения;
- сохраняет исходные материалы, послужившие основанием для появления Work Item;
- не создаёт ложного впечатления полноты при частичном сохранении исходного обсуждения.

---

## Статус

Настоящая спецификация является рабочей.

Её практическая ценность и окончательная структура будут уточняться по мере накопления опыта использования Work Items.

До формирования полноценной спецификации эталонной реализацией считается документ `chatgpt_prompt.md`.
